The problem you already feel

A flat skill folder is a forest with no paths.

~/.claude/skills auto-loads one layer, and a .claude nested inside another .claude never loads. At ten skills that's fine; at a hundred it's melt — the whole pile is in context before you ask anything. The platform ships the nodes and forbids the edge.

skilltree uses the three levers the platform leaves open — nested plain-dir placement (the path is the coordinate), the inert-nested-.claude boundary (only the top auto-loads), and breadcrumbs — to render a real tree you descend on demand.

The front half

Discover. Cohere. Emit.
And undo it exactly.

Point it at any flat .claude/skills. It reads the tree that's actually there, reports where it drifted, and tree-ifies it in place — moving each skill whole (every file kept), journaling every move so unemit restores it byte-for-byte.

→ Nothing is destroyed. Every change is a move with a receipt.

skilltree cohere ~/.claude
# ⚠ bare_forest: 132 top skills, no root
skilltree emit ~/.claude --root-forest
# tree-ified, lossless, journaled
skilltree unemit ~/.claude
# ...and back, byte-for-byte

Verified against the runtime — not asserted

Every load-bearing fact here was observed, not guessed.

skilltree rests on one fact about how Claude Code loads context. We didn't take it on faith — we planted a marker skill, accessed files different ways, and watched the model's tool menu change. The substrate is the oracle.

  • Read a file in a project dir → its CLAUDE.md + rules + skills inject — dynamically, one layer. (Descent is real and in-session.)
  • A .claude nested below the dir you're in does not load — the flatness is enforced; depth is invisible until entered.
  • The trigger is the Read tool, not cat — a shell cat of the same file injects nothing. (That's why the breadcrumbs say "Read," not "cat" — a literal cat would silently fail to descend.)

Those three facts are the whole mechanism — the disease (flat melts) and the cure (a tree you Read down). How they were found, and the bug they caught along the way, is the Dev Log.

It keeps itself honest

A tree that maintains its own coherence.

A structure the platform won't maintain decays back to the forest it replaced — so skilltree watches itself. A scheduled check writes its verdict into a single managed rule, so a decohered tree announces itself in every session: "you drifted — here's the shape, recohere it."

→ Detection is automatic; the fix is one command (emit).

skilltree watch ~/.claude --interval 300
# every 5 min: cohere → rewrite
# ~/.claude/rules/00-system-notifications
# "None. Systems nominal." ⇄ 🔴 ERROR
